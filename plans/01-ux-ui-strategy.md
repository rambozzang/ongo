## 1. 현황 분석

### 1.1 Publer 약점 (유저 리뷰 기반)

| 약점 | 심각도 | onGo 공략 포인트 |
|------|--------|-----------------|
| "Interface feels 5 years old" | 치명적 | 모던 글래스모피즘 + 마이크로 인터랙션으로 시각적 차별화 |
| 복잡한 네비게이션 | 높음 | 3-depth 이하 플랫 구조, 커맨드 팔레트(Cmd+K) 도입 |
| 학습 곡선 높음 | 높음 | 프로그레시브 디스클로저 + 인라인 코칭 |
| 모바일 앱 부실 | 높음 | 네이티브급 PWA + 제스처 기반 모바일 UX |
| 디자인 세련미 부족 | 중간 | 일관된 디자인 토큰 + 타이포그래피 시스템 |
| UX 평점 3.5/5 | 기준 | 목표: 4.5/5 이상 |

### 1.2 onGo 현재 상태 진단

**잘 되어 있는 것:**
- Vue 3 + Tailwind CSS 기반의 견고한 기술 스택
- 다크/라이트/시스템 모드 3종 지원 (`stores/theme.ts`)
- 사이드바 + 탑바 + 모바일 바텀 네비게이션 기본 레이아웃 갖춤
- HeadlessUI 활용으로 접근성 기초 확보
- 컴포넌트화 잘 되어있음 (common/dashboard/upload/layout 분리)
- 반응형 breakpoint 3단계 정의 (mobile: 480px, tablet: 768px, desktop: 1280px)

**개선이 필요한 것:**
- 디자인 토큰이 `main.css`의 Tailwind `@apply` 수준에 머물러 있음 — 체계적 토큰 시스템 부재
- 마이크로 인터랙션이 거의 없음 (Toast 트랜지션 정도가 전부)
- 대시보드가 단순 KPI 카드 + 차트 나열 — 감동이 없음
- 모바일 바텀 네비에 5개 항목만 노출, 나머지 기능 접근 불편
- 온보딩이 3단계 위저드이나 가치 전달(Aha moment) 단계가 3단계(AI 체험)로 너무 늦음
- 로딩 상태가 단순 스피너(`LoadingSpinner.vue`) — 스켈레톤 UI 없음
- 커맨드 팔레트 없음 — 파워 유저 배려 부족
- 타이포그래피 스케일이 정의되어 있지 않음
- 색상이 Violet/Purple 단일 계열 — 의미 기반 색상 시스템 부재

## 2. 디자인 시스템 강화

> 목표: Notion/Linear 수준의 정제된 디자인 시스템. 일관성과 확장성을 동시에 확보.

### 2.1 디자인 토큰 시스템 구축 [P0]

**현재 문제:** `tailwind.config.js`에 primary 색상만 정의, `main.css`에 `.btn-primary`, `.card` 등 산발적 유틸리티 클래스.

**구현 방안:**

CSS Custom Properties 기반 디자인 토큰 계층 구조를 도입한다.

```
tokens/
  primitive.css      -- 원시값 (색상 팔레트, 폰트 크기, 간격)
  semantic.css       -- 의미값 (--color-surface, --color-text-primary, --color-border)
  component.css      -- 컴포넌트 토큰 (--button-height-md, --card-radius)
```

### 2.2 타이포그래피 스케일 [P0]

**현재 문제:** 각 컴포넌트마다 `text-2xl font-bold`, `text-sm font-medium` 등을 임의로 조합. 일관성 없음.

**구현 방안:**

Tailwind 커스텀 유틸리티로 타이포그래피 프리셋을 정의한다. 기기 크기에 따라 유동적 크기를 적용한다.

```
Display:    32px / 40px line-height / -0.02em / Bold     -- 페이지 히어로
Heading 1:  24px / 32px / -0.01em / Bold                 -- 페이지 제목
Heading 2:  20px / 28px / -0.01em / Semibold             -- 섹션 제목
Heading 3:  16px / 24px / 0 / Semibold                   -- 카드 제목
Body:       14px / 22px / 0 / Regular                    -- 본문
Body SM:    13px / 20px / 0 / Regular                    -- 보조 텍스트
Caption:    12px / 16px / 0.01em / Medium                -- 라벨, 배지
Overline:   11px / 16px / 0.05em / Semibold Uppercase    -- 섹션 오버라인
```

**모바일 스케일 조정:** Display는 28px, H1은 22px으로 한 단계씩 축소. 나머지 동일.

**폰트 페어링:**
- 한국어: Pretendard (가변 폰트, 현대적이고 가독성 높음)
- 영문/숫자: Inter (Linear/Vercel이 사용하는 기하학적 산세리프)
- 코드/숫자 강조: JetBrains Mono 또는 Fira Code (모노스페이스)

**예상 효과:**
- 한국어 크리에이터에게 친숙하면서도 글로벌 SaaS 느낌
- 정보 위계(hierarchy) 명확화 — 시선 흐름 자연스러워짐
- Publer의 "올드한 느낌" 대비 확실한 모던 인상

### 2.3 컴포넌트 라이브러리 고도화 [P0]

**현재 문제:** `main.css`에 6개 유틸리티 클래스(btn-primary/secondary/danger, input-field, card, badge 변형)가 전부. Vue 컴포넌트 수준의 디자인 시스템 부재.

**구현 방안:**

체계적 컴포넌트 라이브러리를 구축한다. 각 컴포넌트는 사이즈 변형(sm/md/lg), 색상 변형(variant), 상태(loading/disabled/error)를 Props로 지원한다.

```
components/
  ui/
    OButton.vue          -- variant: primary/secondary/ghost/danger, size: sm/md/lg
    OInput.vue           -- type, prefix/suffix slot, error state, clearable
    OTextarea.vue        -- auto-resize, character count
    OSelect.vue          -- searchable, multi-select, custom render
    OBadge.vue           -- color, size, dot indicator
    OAvatar.vue          -- image/initials/icon, size, online indicator
    OCard.vue            -- variant: default/elevated/bordered/interactive
    OModal.vue           -- size, slide-over variant, nested support
    ODropdown.vue        -- command menu 스타일, keyboard navigation
    OTooltip.vue         -- placement, delay, rich content
    OTabs.vue            -- line/pill/enclosed variant, with icons
    OTag.vue             -- closable, with icon, color variants
    OSwitch.vue          -- with label, description
    OProgress.vue        -- linear/circular, with label
    OEmptyState.vue      -- icon, title, description, action
    OSkeleton.vue        -- text/circle/rect/card preset shapes
    OCommandPalette.vue  -- Cmd+K 글로벌 커맨드 팔레트
```

**핵심: OCommandPalette (Cmd+K 커맨드 팔레트)** [P1]

Linear/Notion의 킬러 UX. 키보드만으로 모든 기능에 접근할 수 있게 한다.

```
기능:
- Cmd+K (Mac) / Ctrl+K (Windows)로 호출
- 퍼지 검색으로 모든 페이지/액션/영상 즉시 접근
- 최근 접근 항목 자동 노출
- 섹션 분리: 네비게이션 / 액션 / 영상 검색 / 설정

액션 목록:
- "새 영상 업로드" → /upload 이동
- "대시보드" → /dashboard 이동
- "다크모드 전환" → 테마 즉시 토글
- "AI 제목 생성" → /ai 이동
- "채널 관리" → /channels 이동
- 영상 제목 검색 → 해당 영상 상세 페이지
```

**예상 효과:**
- 파워 유저 생산성 2배 향상
- "이 앱 진짜 잘 만들었다" 첫인상 — SaaS 경험이 있는 유저 즉시 인지
- Publer에 없는 기능으로 차별화 극대화

### 2.4 색상 시스템 정교화 [P1]

**현재 문제:** 플랫폼 색상(youtube/tiktok/instagram/naver)은 정의되어 있으나 의미 기반 색상 체계 부재.

**구현 방안:**

```
Accent 색상 체계:
- Brand Primary: Violet-600 (#7c3aed) -- 현재 유지
- Brand Gradient: Violet-600 → Purple-500 -- CTA 버튼, 프리미엄 강조
- AI 특화: Violet-500 → Fuchsia-500 그라디언트 -- AI 관련 UI 전용

상태 색상 (현재 badge에만 적용, 전역 확장):
- Success: Emerald-500 (#10b981) -- 게시 완료, 연동 성공
- Warning: Amber-500 (#f59e0b) -- 크레딧 부족 경고, 검토 중
- Error: Red-500 (#ef4444) -- 업로드 실패, 거부됨
- Info: Blue-500 (#3b82f6) -- 새 기능 안내, 팁

플랫폼 색상 (기존 유지 + alpha 변형 추가):
- YouTube: #FF0000 + bg: rgba(255,0,0,0.08)
- TikTok: #000000 (light) / #E5E7EB (dark) + bg 변형
- Instagram: 그라디언트 (#833AB4 → #FD1D1D → #F77737)
- Naver Clip: #03C75A + bg 변형

데이터 시각화 전용 팔레트:
- 4색: #7c3aed, #06b6d4, #f59e0b, #10b981
- 보조 4색: #ec4899, #8b5cf6, #14b8a6, #f97316
```

**예상 효과:**
- 색상만으로 정보 유형 즉시 파악 가능
- 플랫폼 아이덴티티를 일관되게 유지하면서 onGo 브랜드와 조화
- 차트/그래프의 가독성 대폭 향상

### 2.5 Glassmorphism + Subtle Depth [P1]

**현재 문제:** 카드가 단순 `border + shadow-sm`. 깊이감 없이 평면적.

**구현 방안:**

모던 SaaS에서 유행하는 유리 효과(glassmorphism)를 사이드바, 모달, 드롭다운에 적용한다. 과하지 않게, 배경 블러 + 반투명도만 살짝 사용한다.

```css
/* 사이드바 */
.glass-sidebar {
  background: rgba(255, 255, 255, 0.8);
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  border-right: 1px solid rgba(0, 0, 0, 0.06);
}

.dark .glass-sidebar {
  background: rgba(15, 23, 42, 0.85);
  border-right: 1px solid rgba(255, 255, 255, 0.06);
}

/* 떠있는 카드 (모달, 드롭다운) */
.glass-elevated {
  background: rgba(255, 255, 255, 0.9);
  backdrop-filter: blur(16px);
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.08),
              0 0 0 1px rgba(0, 0, 0, 0.04);
}

/* 카드 호버 시 미묘한 상승 효과 */
.card-interactive {
  transition: transform 200ms ease, box-shadow 200ms ease;
}
.card-interactive:hover {
  transform: translateY(-2px);
  box-shadow: 0 12px 24px rgba(0, 0, 0, 0.1);
}
```

**예상 효과:**
- "와, 깔끔하다" 첫인상 — Publer의 "5년 전 인터페이스"와 극명한 대비
- 콘텐츠 계층 구조 시각적으로 명확
- 과하지 않은 수준으로 성능 부담 최소화

## 3. 마이크로 인터랙션

> 목표: 모든 사용자 액션에 즉각적이고 유쾌한 피드백. "무미건조한 도구"에서 "쓰는 맛이 있는 앱"으로.

### 3.1 페이지 전환 애니메이션 [P0]

**현재 문제:** `<router-view />`가 날것으로 교체됨. 페이지 이동 시 깜빡임.

**구현 방안:**

Vue `<Transition>` 래퍼를 `<router-view>`에 적용한다.

```vue
<!-- AppLayout.vue 내부 -->
<router-view v-slot="{ Component, route }">
  <Transition
    :name="transitionName"
    mode="out-in"
    @before-enter="onBeforeEnter"
  >
    <component :is="Component" :key="route.path" />
  </Transition>
</router-view>
```

전환 효과 종류:
- **기본 (페이지 간):** fade + 약간의 slideUp (opacity 0→1, translateY 8px→0, 200ms)
- **드릴다운 (목록→상세):** slide-left (현재 왼쪽으로 사라지고 새 페이지 오른쪽에서 등장)
- **모달형 전환:** scale + fade (중앙에서 팝업)

**우선순위:** P0
**예상 효과:** 페이지 이동이 부드러워져 앱 느낌 극대화. 웹사이트가 아닌 "애플리케이션"으로 인식.

### 3.2 스켈레톤 로딩 UI [P0]

**현재 문제:** `LoadingSpinner.vue`가 단순 회전 원. 콘텐츠 로딩 중 빈 화면 + 스피너로 CLS(Cumulative Layout Shift) 발생.

**구현 방안:**

`OSkeleton.vue` 컴포넌트를 만들어 각 페이지별 스켈레톤 프리셋을 제공한다.

```
적용 대상:
- DashboardView: SummaryCard 4개 스켈레톤 + 차트 영역 스켈레톤 + 리스트 스켈레톤
- VideosView: 비디오 카드 그리드 스켈레톤
- AnalyticsView: 차트 + 테이블 스켈레톤
- ChannelsView: 채널 카드 스켈레톤

스켈레톤 애니메이션:
- shimmer 효과 (좌→우 빛 줄무늬 이동)
- 다크 모드에서는 밝은 shimmer → 어두운 shimmer 전환
- 실제 콘텐츠와 동일한 레이아웃 유지하여 CLS 제로화
```

**우선순위:** P0
**예상 효과:**
- 체감 로딩 속도 40% 향상 (실제 속도 변화 없이 인지 속도 개선)
- 전문적인 앱 인상 강화

### 3.3 버튼 및 인터랙티브 요소 피드백 [P0]

**현재 문제:** 버튼은 `hover:bg-xxx`만 존재. 클릭 시 시각적 피드백 없음.

**구현 방안:**

```css
/* Press feedback (모든 인터랙티브 요소) */
.btn-press {
  transition: transform var(--transition-fast), box-shadow var(--transition-fast);
}
.btn-press:active {
  transform: scale(0.97);
}

/* Ripple effect (선택적 — Material Design 영감) */
/* 대신 onGo는 더 미니멀한 접근: active시 subtle scale */

/* 포커스 링 개선 (현재: ring-2 ring-primary-500) */
/* 개선: offset + glow 효과 */
.focus-ring {
  outline: none;
  box-shadow: 0 0 0 2px var(--surface-primary),
              0 0 0 4px var(--accent-primary);
}
```

**추가 인터랙션:**
- 스위치 토글: 탭 시 spring 애니메이션 (cubic-bezier(0.34, 1.56, 0.64, 1))
- 체크박스: 체크 시 체크마크가 그려지는 SVG path 애니메이션
- 드롭다운: 아래로 펼쳐질 때 origin-top + scaleY 애니메이션

**우선순위:** P0
**예상 효과:** 모든 클릭에 반응하는 앱 — "터치감"이 좋은 UX

### 3.4 드래그 & 드롭 강화 [P1]

**현재 문제:** `FileDropzone.vue`에 기본 drag/drop만 구현. 시각적 피드백 빈약.

**구현 방안:**

```
파일 드롭존 개선:
1. 드래그 진입 시: 보더가 실선으로 변하면서 펄스 애니메이션
2. 드래그 중: 드롭 영역이 살짝 확대(scale 1.01) + 배경색 그라디언트 전환
3. 드롭 시: 체크마크 lottie 애니메이션 + 파일명 표시
4. 다중 파일: 겹쳐진 카드 프리뷰

스케줄 캘린더 드래그:
1. 예약 영상을 캘린더 셀 간 드래그로 일정 변경
2. 드래그 중 고스트 프리뷰 + 타임슬롯 하이라이트
3. 드롭 시 스냅 애니메이션 + 확인 토스트

영상 목록 정렬:
1. 영상 카드 순서 드래그 변경
2. 드래그 시 카드 tilting(기울어짐) 효과 + shadow 증가
3. placeholder가 점선으로 표시
```

**기술:** `@vueuse/core`의 `useDraggable` 활용 또는 `vuedraggable`(SortableJS 래퍼) 사용.

**우선순위:** P1
**예상 효과:** "와, 이것도 되네?" — 기대 이상의 인터랙션으로 만족도 상승

### 3.5 숫자 카운트업 애니메이션 [P1]

**현재 문제:** `SummaryCard.vue`에서 `formattedValue`가 즉시 표시. 대시보드 진입 시 밋밋.

**구현 방안:**

`useCountUp` composable을 만든다.

```
동작:
- 대시보드 진입 시 0 → 실제 값으로 카운트업
- duration: 800ms, easing: ease-out
- 천 단위 콤마가 실시간으로 갱신
- K/M 축약도 자연스럽게 전환 (999 → 1.0K)

추가:
- 변화율 표시도 딜레이 후 fade-in (카운트업 완료 후 200ms)
- AI 크레딧 게이지바: 좌→우 fill 애니메이션
```

**우선순위:** P1
**예상 효과:** 대시보드가 "살아 움직이는" 느낌. 숫자가 커지는 것을 보며 성취감 자극.

### 3.6 Toast 및 알림 개선 [P1]

**현재 문제:** `ToastNotification.vue`가 오른쪽에서 슬라이드인만 있음. 닫힘 시 역방향 슬라이드.

**구현 방안:**

```
개선 사항:
1. 성공 토스트: 좌측에 작은 체크 애니메이션 (원 그리기 → 체크마크 그리기)
2. 에러 토스트: 좌측에 흔들림(shake) 아이콘
3. 진행 상태 토스트: 프로그레스 바 하단 표시 (업로드 진행률)
4. 스택형: 여러 토스트가 쌓일 때 뒤쪽 토스트 축소 (1→0.95→0.9 scale)
5. 스와이프로 닫기 (모바일)
6. 자동 닫힘 타이머 시각화 (하단에 타이머 바)
```

**우선순위:** P1
**예상 효과:** 피드백의 질적 향상. 성공/실패를 아이콘 + 애니메이션으로 즉시 인지.

### 3.7 AI 처리 중 특화 애니메이션 [P1]

**현재 문제:** AI 생성 시 단순 스피너 (`animate-spin`).

**구현 방안:**

AI는 onGo의 핵심 차별화 요소이므로 전용 로딩 UX를 설계한다.

```
AI 생성 중 UI:
1. 보라색 그라디언트 펄스 배경 (Violet → Purple → Fuchsia 순환)
2. "AI가 분석 중입니다..." → "제목을 생성하고 있습니다..." → "태그를 최적화하고 있습니다..." 단계별 메시지 전환
3. 타이핑 효과로 결과 한 글자씩 나타남 (ChatGPT 스트리밍 느낌)
4. 생성 완료 시 confetti 또는 sparkle 파티클 효과 (3초간)
5. 생성된 항목들이 하나씩 stagger 방식으로 fade-in

기술:
- CSS @keyframes로 그라디언트 애니메이션
- requestAnimationFrame 기반 타이핑 효과
- <canvas> 또는 CSS 파티클로 축하 효과
```

**우선순위:** P1
**예상 효과:** AI 대기 시간이 "지루한 기다림"에서 "흥미로운 과정"으로 전환. AI 기능 사용률 증가 기대.

## 4. 대시보드 UX 혁신

> 목표: 대시보드를 열었을 때 "내 채널 상태가 한눈에" + "다음에 뭘 해야 하는지 즉시 파악" + 시각적 임팩트.

### 4.1 개인화 인사 및 오늘의 요약 헤더 [P0]

**현재 문제:** `<h1>대시보드</h1>`가 전부. 개인화 없음.

**구현 방안:**

```
헤더 레이아웃:
┌─────────────────────────────────────────────────────┐
│ 좋은 아침이에요, 범규님!                    [오늘 2/9] │
│ 오늘 1개 영상이 예약되어 있어요 · AI 크레딧 127개 남음  │
│                                                      │
│ [+ 새 영상 업로드]  [AI 제목 생성]  [일정 확인]         │
└─────────────────────────────────────────────────────┘

인사 로직:
- 06:00~11:59 → "좋은 아침이에요"
- 12:00~17:59 → "좋은 오후예요"
- 18:00~21:59 → "좋은 저녁이에요"
- 22:00~05:59 → "밤늦게까지 수고하세요"

Quick Action 버튼:
- 핵심 3가지 액션 노출 (사용 빈도 기반 동적 변경)
- 호버 시 간략 설명 툴팁
```

**우선순위:** P0
**예상 효과:** 로그인 즉시 "나를 위한 대시보드" 느낌. Publer의 제네릭 대시보드 대비 개인화 차별.

### 4.2 KPI 카드 비주얼 업그레이드 [P0]

**현재 문제:** `SummaryCard.vue`가 단순 숫자 + 아이콘 + 변화율. 시각적 임팩트 부족.

**구현 방안:**

```
개선 포인트:
1. 각 카드에 배경 그라디언트 아이콘 영역 추가
   - 조회수: Indigo 그라디언트 + 눈 아이콘
   - 구독자: Emerald 그라디언트 + 사람 아이콘
   - 좋아요: Rose 그라디언트 + 하트 아이콘
   - AI 크레딧: Violet → Purple 그라디언트 + 스파크 아이콘

2. 숫자 카운트업 애니메이션 (3.5절 연동)

3. 변화율 뱃지를 화살표 → 미니 스파크라인(sparkline)으로 교체
   - 최근 7일 트렌드를 작은 선 그래프로 카드 내부에 표시
   - 상승 시 초록, 하락 시 빨강

4. 호버 시 카드가 살짝 올라오면서 그림자 깊어짐 (3.3절 연동)

5. 클릭 가능 상태 시각화 (현재 cursor-pointer만)
   - 우하단에 작은 화살표 아이콘 + "자세히 보기" 텍스트 (호버 시 나타남)
```

**우선순위:** P0
**예상 효과:** 대시보드의 "와!" 팩터. KPI 카드가 그 자체로 미니 인포그래픽 역할.

### 4.3 실시간 업로드/게시 상태 피드 [P0]

**현재 문제:** 현재 대시보드에는 `RecentVideosList`(최근 영상)와 `UpcomingSchedules`(예약)만 있음. 진행 중인 작업 상태를 확인하려면 다른 페이지로 이동해야 함.

**구현 방안:**

대시보드 상단(KPI 카드 아래)에 "Activity Feed" 영역을 추가한다.

```
┌─ 진행 중 ──────────────────────────────────────────┐
│ [====>    ] 뷰티 브이로그.mp4 업로드 중 (67%)       │
│ [YouTube ✓] [TikTok ⟳] [Instagram ⟳] 여행 영상 게시 중│
│ [예약] 2/10 오후 3:00 — "겨울 코디 추천" 게시 예정    │
└──────────────────────────────────────────────────────┘

특징:
- 실시간 진행률 바 (WebSocket 또는 폴링)
- 플랫폼별 상태 아이콘 (체크/로딩/실패)
- 없을 때: "모든 작업이 완료되었어요!" + 업로드 CTA
- 최대 3건까지 표시, 더보기 클릭 시 영상 관리 페이지
- 각 항목 클릭 시 해당 영상 상세로 이동
```

**우선순위:** P0
**예상 효과:** 대시보드 하나로 현재 진행 상황 전부 파악. 페이지 이동 빈도 감소. Publer에 없는 실시간 피드.

### 4.4 차트 인터랙션 개선 [P1]

**현재 문제:** `TrendChart.vue`와 `PlatformPieChart.vue`가 Chart.js 기본 설정. 인터랙션 빈약.

**구현 방안:**

```
TrendChart 개선:
1. 호버 시 크로스헤어 라인 + 하이라이트 포인트 확대
2. 범례 클릭 시 해당 플랫폼 라인만 강조 (나머지 dimmed)
3. 차트 위 "베스트 일자" 하이라이트 (가장 높은 포인트에 작은 배지)
4. 기간 전환 시 부드러운 모프(morph) 애니메이션
5. 빈 데이터: 아직 데이터가 없는 플랫폼은 점선으로 표시

PlatformPieChart 개선:
1. 파이 → 도넛 차트로 변경 (중앙에 총합 표시)
2. 호버 시 해당 세그먼트 확대 + 상세 수치 팝업
3. 애니메이션: 진입 시 각 세그먼트가 순서대로 나타남 (stagger)
4. 빈 상태: 연동된 플랫폼 아이콘 + "아직 데이터를 수집 중이에요"

기술: Chart.js 커스텀 플러그인 또는 더 인터랙티브한 라이브러리(ECharts/ApexCharts) 검토
```

**우선순위:** P1
**예상 효과:** 데이터 분석이 재미있는 경험으로. 크리에이터가 대시보드를 자주 방문하게 됨.

### 4.5 위젯 커스터마이제이션 [P2]

**현재 문제:** 대시보드 레이아웃이 고정. 사용자마다 다른 KPI에 관심.

**구현 방안:**

```
기능:
1. 위젯 추가/제거/재배열 (드래그 & 드롭)
2. 위젯 사이즈 조절 (1x1, 2x1, 2x2)
3. 사용 가능한 위젯:
   - KPI 카드 (조회수/구독자/좋아요/댓글/공유)
   - 트렌드 차트 (기간별)
   - 플랫폼 비교
   - 최근 영상
   - 예약 일정
   - AI 크레딧 상태
   - 인기 태그 클라우드
   - 채널 건강도 스코어
4. 레이아웃 저장/로드 (localStorage → 추후 서버 동기화)
5. 프리셋 레이아웃: "콘텐츠 크리에이터" / "데이터 분석가" / "심플"

기술: CSS Grid + vue-grid-layout 또는 직접 구현
```

**우선순위:** P2
**예상 효과:** "내 대시보드"라는 소유감. Pro/Business 플랜 차별화 요소로 활용 가능.

## 5. 모바일 퍼스트 전략

> 목표: 크리에이터는 이동 중에 알림 확인, 댓글 확인, 간단한 편집을 한다. 모바일에서의 핵심 플로우가 데스크톱만큼 자연스러워야 한다.

### 5.1 바텀 시트 패턴 도입 [P0]

**현재 문제:** 모바일에서 모달이 화면 중앙에 뜸. 엄지로 닫기 어려움.

**구현 방안:**

모바일에서 모든 모달/드롭다운을 바텀 시트로 대체한다.

```
바텀 시트 컴포넌트 (OBottomSheet.vue):
- 아래에서 위로 슬라이드 업
- 상단에 드래그 핸들 (회색 bar)
- 아래로 스와이프하여 닫기 (속도 감지 — 빠르게 내리면 즉시 닫힘)
- 3단계 높이: peek(25%) / half(50%) / full(90%)
- 스와이프 속도에 따른 스냅 동작
- 뒤 배경 dim + 터치 시 닫기

적용 대상:
- 영상 상세 정보 (목록에서 탭)
- 필터/정렬 옵션
- 플랫폼 선택
- 프로필 메뉴 (현재: 드롭다운)
- 테마 선택
- 알림 목록
```

**우선순위:** P0
**예상 효과:** iOS/Android 네이티브 앱 수준의 모바일 UX. Publer 모바일 앱 대비 압도적 터치 경험.

### 5.2 모바일 네비게이션 재설계 [P0]

**현재 문제:** `MobileBottomNav.vue`에 5개 항목(대시보드/업로드/예약/분석/더보기)만 노출. 영상 관리, AI 도구, 채널 관리는 "더보기"에 숨겨짐.

**구현 방안:**

```
바텀 네비 개선:
┌─────────────────────────────────────┐
│ [홈]  [업로드+]  [영상]  [분석]  [더보기]│
└─────────────────────────────────────┘

변경 사항:
1. "예약" → "영상"으로 변경 (더 자주 접근하는 기능)
2. "업로드" 아이콘을 중앙에 크게 (FAB 스타일, 배경 그라디언트)
   - 핵심 CTA를 시각적으로 강조
   - 탭 시 ripple 효과
3. "더보기" 탭 시 바텀 시트로 전체 메뉴 노출:
   - 예약 관리
   - AI 도구 (스파크 아이콘 + "NEW" 뱃지)
   - 채널 관리
   - 구독/결제
   - 설정
4. 현재 활성 탭: 아이콘 색상 변경 + 라벨 표시 + 아이콘 위 작은 dot
5. 스크롤 다운 시 바텀 네비 숨김 → 스크롤 업 시 나타남 (auto-hide)

기술: Intersection Observer로 스크롤 방향 감지
```

**우선순위:** P0
**예상 효과:** 모바일 핵심 동선 간소화. 업로드 CTA가 항상 시각적으로 두드러짐.

### 5.3 모바일 업로드 플로우 최적화 [P0]

**현재 문제:** 업로드 3단계(파일→메타→플랫폼)가 데스크톱과 동일. 모바일에서 입력 필드가 많으면 이탈.

**구현 방안:**

```
모바일 전용 간편 업로드 플로우:
1. 바텀 네비 "+" 탭 → 카메라 롤 열기 OR 촬영 (네이티브 <input capture>)
2. 파일 선택 즉시 업로드 시작 (Tus 프로토콜)
3. 업로드 중에 제목 입력 (필수 1개만 요구)
4. "AI 자동 완성" 원탭 버튼 — 제목/설명/태그 자동 생성
5. 플랫폼 선택은 토글 스위치 (이전 설정 기억)
6. "즉시 게시" 원탭 — 끝

총 탭 수: 4~5회 (파일선택 → 제목입력 → AI자동완성 → 게시)

데스크톱 풀 플로우는 유지하되, 모바일에서는 이 간편 플로우가 기본.
```

**우선순위:** P0
**예상 효과:** 모바일 업로드 완료 시간 80% 단축. "이동 중에도 영상 올리기" 핵심 가치 실현.

### 5.4 제스처 기반 인터랙션 [P1]

**현재 문제:** 모바일에서 모든 상호작용이 "탭" 하나. 제스처 활용 없음.

**구현 방안:**

```
제스처 매핑:
- 왼쪽 스와이프 (영상 카드): 삭제 옵션 노출 (빨간 배경 + 휴지통 아이콘)
- 오른쪽 스와이프 (영상 카드): 빠른 공유/복제 옵션
- 아래로 당기기 (Pull-to-refresh): 대시보드/영상 목록 새로고침
  - 커스텀 로딩 인디케이터 (onGo 로고 회전)
- 핀치 줌 (분석 차트): 기간 확대/축소
- 길게 누르기 (영상 카드): 컨텍스트 메뉴 바텀시트
  - 햅틱 피드백 (navigator.vibrate)

기술: @vueuse/gesture 또는 Hammer.js
```

**우선순위:** P1
**예상 효과:** 네이티브 앱과 동등한 터치 경험. 모바일 사용 효율 대폭 향상.

### 5.5 PWA 강화 [P1]

**현재 문제:** PWA 설정 없음. 홈 화면 추가 불가.

**구현 방안:**

```
PWA 매니페스트:
- name: "onGo - 크리에이터 플랫폼"
- short_name: "onGo"
- display: "standalone"
- orientation: "portrait"
- theme_color: "#7c3aed"
- background_color: "#ffffff"
- 아이콘: 512x512, 192x192, maskable

서비스 워커:
- Workbox 기반 (vite-plugin-pwa)
- 전략: NetworkFirst (API) + CacheFirst (정적 자산)
- 오프라인 폴백 페이지: "인터넷 연결을 확인해주세요" + 캐시된 대시보드 데이터 표시

푸시 알림:
- 게시 완료 알림
- 예약 영상 게시 알림
- AI 크레딧 부족 경고
- 채널 연동 만료 알림

기술: vite-plugin-pwa + Firebase Cloud Messaging (또는 Web Push API 직접)
```

**우선순위:** P1
**예상 효과:** 홈 화면 아이콘 → 앱 실행 경험. 푸시 알림으로 재방문율 30% 향상 기대.

### 5.6 모바일 전용 대시보드 레이아웃 [P1]

**현재 문제:** 대시보드가 데스크톱 레이아웃을 단순히 세로로 스택한 형태. 모바일에서 스크롤 과다.

**구현 방안:**

```
모바일 대시보드 레이아웃:
1. 인사 헤더 (간략화 — 1줄)
2. 진행 중 작업 (있을 경우만, 스와이프 가능 카드)
3. KPI 요약: 2x2 그리드 (작은 카드, 숫자 강조)
4. Quick Action 버튼 3개 (가로 스크롤)
5. 최근 영상 (가로 스크롤 카드, 썸네일 크게)
6. 트렌드 차트 (간소화 — 터치 인터랙션 강화)
7. 예약 일정 (오늘/내일만 표시)

차이점:
- 데스크톱: 그리드 레이아웃 (2열, 3열)
- 모바일: 단일 열 + 가로 스크롤 카드
- 불필요한 정보 숨기고 핵심만 노출
- 섹션마다 "더보기 >" 버튼으로 상세 페이지 연결
```

**우선순위:** P1
**예상 효과:** 모바일에서 3스크롤 이내에 핵심 정보 전부 확인. 정보 과부하 방지.

## 6. 온보딩 UX

> 목표: 첫 사용자가 3분 안에 "이 서비스 진짜 좋다"를 느끼게 하는 Aha moment 설계.

### 6.1 온보딩 플로우 재설계 [P0]

**현재 문제:** 현재 3단계: 프로필 설정(1) → 채널 연동(2) → AI 체험(3). Aha moment인 AI 체험이 마지막에 위치하여, 이전 단계에서 이탈 가능성 높음.

**새로운 온보딩 플로우:**

```
[Welcome Screen] ← NEW
  "onGo에 오신 것을 환영합니다!"
  15초 애니메이션으로 핵심 가치 3가지 시각화:
  1. 영상 하나 → 4개 플랫폼 동시 게시 (파일이 복제되는 애니메이션)
  2. AI가 제목/태그 자동 생성 (타이핑 애니메이션)
  3. 통합 분석 대시보드 (차트가 그려지는 애니메이션)
  "시작하기" 버튼

[Step 1: 프로필] — 현재와 동일하되 간소화
  - 닉네임만 필수 (카테고리는 선택적, 나중에 설정 가능)
  - "30초면 끝나요" 안내 문구

[Step 2: AI 체험] ← 순서 변경! 채널 연동 전에 AI 먼저
  - "AI가 어떤 걸 할 수 있는지 체험해보세요"
  - 샘플 영상 선택지 제공 (뷰티/게임/일상/먹방 중 1개 선택)
  - 선택 즉시 AI 생성 시작 → 결과가 타이핑 효과로 나타남
  - "이 기능을 내 영상에도 쓸 수 있어요!" CTA

[Step 3: 채널 연동] — Aha moment 이후이므로 동기부여 극대화
  - "이제 내 채널을 연결하고 실제로 사용해보세요"
  - 채널 연동 시 해당 채널의 실제 데이터 미리보기 (구독자 수 등)
  - 건너뛰기 가능 (나중에 설정에서)

[Completion]
  - 축하 애니메이션 (confetti)
  - "첫 영상 업로드하기" CTA (가장 큰 버튼)
  - "대시보드 둘러보기" 링크
```

**핵심 변경: AI 체험(Aha moment)을 Step 2로 앞당김.**

**우선순위:** P0
**예상 효과:**
- 온보딩 완료율 40% 향상 (Aha moment 조기 제공으로 이탈 감소)
- "이 앱 진짜 다르다" 첫인상 각인
- 채널 연동 동기 강화 (AI 경험 후 "이걸 내 채널에도 쓰고 싶다")

### 6.2 프로그레시브 디스클로저 (인라인 코칭) [P0]

**현재 문제:** 온보딩 완료 후 대시보드에 떨어지면 어디부터 시작해야 하는지 막막함.

**구현 방안:**

```
첫 방문 시 인라인 코칭 (비침투적):
1. 대시보드 진입 시 → 상단에 "시작 가이드" 배너
   ┌──────────────────────────────────────────────┐
   │ [1/4 완료] onGo 시작하기                       │
   │ [ ] 첫 영상 업로드    [→]                       │
   │ [ ] AI로 제목 최적화  [ ]                       │
   │ [ ] 첫 게시 완료      [ ]                       │
   │ [ ] 분석 확인          [ ]                       │
   │                              [가이드 숨기기]      │
   └──────────────────────────────────────────────┘

2. 각 단계 완료 시 체크 애니메이션 + 다음 단계 하이라이트
3. 4/4 완료 시: "축하합니다! 이제 onGo 마스터입니다!" + 가이드 자동 숨김
4. 진행률 바: 0% → 100% (도파민 자극)

툴팁 코칭 (contextual):
- 업로드 페이지 첫 방문: 드롭존 위에 "여기에 파일을 끌어놓으세요" 애니메이션 가이드
- AI 도구 첫 방문: "크레딧이 있으면 AI가 제목과 태그를 자동 생성해줍니다" 코칭 팝오버
- 분석 페이지 첫 방문: 주요 지표 설명 오버레이

기술:
- localStorage에 각 코칭 표시 여부 저장
- useOnboardingGuide() composable로 상태 관리
- 한 번 닫으면 다시 안 나타남
```

**우선순위:** P0
**예상 효과:**
- 첫 영상 업로드까지의 시간 50% 단축
- 기능 발견률 향상 — "이런 기능도 있었어?"를 방지
- Publer의 "높은 학습 곡선" 대비 매끄러운 사용자 여정

### 6.3 빈 상태(Empty State) 디자인 [P1]

**현재 문제:** `EmptyState.vue`가 존재하지만 기본적인 수준. 각 페이지의 빈 상태가 "데이터 없음" 메시지 정도.

**구현 방안:**

```
페이지별 빈 상태 디자인:

영상 관리 (빈 상태):
  일러스트: 카메라 아이콘 + 점선 프레임
  제목: "아직 영상이 없어요"
  설명: "첫 영상을 업로드하고 모든 플랫폼에 한 번에 게시해보세요."
  CTA: [+ 첫 영상 업로드하기] (primary 버튼)

분석 (빈 상태):
  일러스트: 차트 아이콘 + 물결 선
  제목: "분석할 데이터를 수집 중이에요"
  설명: "영상을 게시하면 24시간 내에 분석 데이터가 나타납니다."
  상태: 연동된 채널 수 표시 + 추가 연동 CTA

채널 관리 (빈 상태):
  일러스트: 연결 아이콘 + 플랫폼 로고 4개
  제목: "채널을 연동하면 시작할 수 있어요"
  설명: "YouTube, TikTok, Instagram, Naver Clip 중 원하는 플랫폼을 연결하세요."
  CTA: [채널 연동하기] + 각 플랫폼 아이콘 버튼

예약 관리 (빈 상태):
  일러스트: 캘린더 + 시계 아이콘
  제목: "예약된 영상이 없어요"
  설명: "업로드 시 예약 게시를 설정하면 이곳에서 관리할 수 있어요."
  CTA: [영상 업로드하기]

공통 원칙:
- 모든 빈 상태에 일러스트 (아이콘 조합 또는 간단한 SVG 일러스트)
- 행동 유도 CTA 반드시 포함
- 설명은 2줄 이내
- 다크 모드에서도 아름답게
```

**우선순위:** P1
**예상 효과:** 빈 상태가 "막다른 길"이 아닌 "다음 액션 안내"로 변환. 첫 사용자 이탈 방지.

### 6.4 인터랙티브 투어 (선택적) [P2]

**현재 문제:** 기능 설명이 온보딩 텍스트에만 의존.

**구현 방안:**

```
기능:
- "가이드 투어 시작" 버튼 (설정 또는 도움말에서 접근)
- 화면 특정 요소를 하이라이트하면서 순서대로 설명
- Step 1: "여기서 영상을 업로드합니다" (업로드 메뉴 하이라이트)
- Step 2: "AI가 제목과 태그를 추천해줍니다" (AI 도구 하이라이트)
- Step 3: "한 번에 여러 플랫폼에 게시합니다" (플랫폼 선택 영역)
- Step 4: "여기서 전체 성과를 확인합니다" (분석 메뉴)
- 각 스텝 사이 부드러운 스크롤 + 하이라이트 이동
- "다음" / "건너뛰기" 버튼

기술: driver.js 또는 직접 구현 (overlay + spotlight 패턴)
```

**우선순위:** P2
**예상 효과:** 복잡한 기능도 쉽게 학습. 고객 지원 문의 감소.

## 7. 접근성 및 국제화

> 목표: 한국어 기본, 영어 확장. WCAG 2.1 AA 기준 준수. 다양한 사용자 환경 대응.

### 7.1 키보드 접근성 강화 [P0]

**현재 문제:** HeadlessUI 덕에 기본적인 접근성은 있으나, 커스텀 컴포넌트(SummaryCard, PlatformSelector 등)에 키보드 지원 부재.

**구현 방안:**

```
키보드 네비게이션 전체 점검:
1. 모든 인터랙티브 요소에 tabindex, role, aria-label 부여
2. 포커스 순서: 왼→오, 위→아래 자연스러운 흐름
3. 커스텀 포커스 링 (2.3절의 focus-ring 디자인 적용)
4. Escape: 모달/드롭다운/바텀시트 닫기
5. Enter/Space: 버튼/링크 활성화
6. Arrow Keys: 카드 그리드 탐색, 탭 전환
7. Tab Trap: 모달 열려있을 때 모달 내부에서만 포커스 순환

스크린 리더 지원:
- SummaryCard: "총 조회수 152만, 전주 대비 12% 상승"
- StatusBadge: "게시 상태: 처리 중"
- 차트: aria-label로 핵심 데이터 포인트 텍스트 제공
- 토스트: role="alert" aria-live="polite"
- 로딩 상태: aria-busy="true"

Skip to content:
- 페이지 최상단에 "본문으로 건너뛰기" 숨겨진 링크
- Tab 시 노출 → 본문 영역으로 즉시 이동
```

**우선순위:** P0
**예상 효과:** WCAG 2.1 AA 기준 충족. 법적 리스크 제거. 다양한 사용자 대응.

### 7.2 색상 대비 및 시각 접근성 [P0]

**현재 문제:** 일부 텍스트(예: `text-gray-400 dark:text-gray-500`)가 배경 대비 4.5:1 미달 가능.

**구현 방안:**

```
점검 및 수정 대상:
1. 모든 본문 텍스트: 배경 대비 4.5:1 이상
2. 대형 텍스트(18px+ 또는 14px+ bold): 3:1 이상
3. 인터랙티브 요소(버튼, 링크): 3:1 이상
4. 비활성 상태: 대비 면제이나 disabled 시각적 표시 명확히

구체적 수정:
- text-gray-400 (본문 보조): → text-gray-500 (light) / text-gray-400 (dark)
- placeholder 색상: → text-gray-500 (현재 text-gray-400으로 대비 부족)
- 배지 텍스트와 배경 대비 점검

도구:
- axe DevTools로 자동 감사
- Chrome Lighthouse 접근성 점수 90+ 목표
```

**우선순위:** P0
**예상 효과:** 시력이 좋지 않은 사용자도 편안하게 사용. 접근성 인증 기반 마련.

### 7.3 국제화(i18n) 아키텍처 [P1]

**현재 문제:** 모든 텍스트가 한국어 하드코딩. 영어 확장 시 전면 수정 필요.

**구현 방안:**

```
기술: vue-i18n (Vue 3 공식 i18n 라이브러리)

디렉토리 구조:
locales/
  ko/
    common.json      -- 공통 (확인, 취소, 저장, 닫기 ...)
    dashboard.json    -- 대시보드 전용
    upload.json       -- 업로드 전용
    auth.json         -- 인증 관련
    ...
  en/
    common.json
    dashboard.json
    ...

번역 키 네이밍 규칙:
- 페이지.섹션.요소 (예: "dashboard.kpi.totalViews")
- CTA: 동사형 (예: "upload.action.publish" → "게시하기" / "Publish")
- 에러: 문장형 (예: "upload.error.fileTooLarge" → "파일 크기가 2GB를 초과합니다.")

숫자/날짜 포맷:
- 한국어: 2026년 2월 9일 (일) / 1,234,567 / ₩9,900
- 영어: Feb 9, 2026 (Sun) / 1,234,567 / $8.99

구현 단계:
Phase 1: i18n 셋업 + 주요 UI 텍스트 추출 (네비게이션, 버튼, 제목)
Phase 2: 전체 텍스트 추출 (에러 메시지, 설명, 빈 상태 등)
Phase 3: 영어 번역 + 언어 전환 UI
Phase 4: RTL 지원 준비 (아랍어 등 확장 대비)
```

**우선순위:** P1
**예상 효과:**
- 영어권 크리에이터 시장 확장 기반 마련
- 글로벌 SaaS로의 성장 가능성 확보
- 번역 관리 체계화

### 7.4 반응형 폰트 및 줌 대응 [P1]

**현재 문제:** 브라우저 기본 폰트 크기 변경 시 레이아웃 깨짐 가능성.

**구현 방안:**

```
원칙:
1. 모든 폰트 크기를 rem 단위로 (이미 Tailwind 기본)
2. 간격(padding/margin)도 rem 기반
3. 200% 줌에서도 레이아웃 정상 동작 확인
4. max-width를 ch 단위로 설정 (가독성 최적화: 본문 65~80ch)
5. 포커스 인디케이터가 줌 시에도 최소 2px 두께 유지

테스트:
- 브라우저 기본 폰트 크기 16px / 20px / 24px에서 레이아웃 확인
- 100% / 150% / 200% 줌에서 기능 정상 동작 확인
- OS 접근성 설정(대형 텍스트, 고대비 모드)에서 확인
```

**우선순위:** P1
**예상 효과:** WCAG 1.4.4 (텍스트 크기 조절) 준수. 다양한 사용자 환경 대응.

### 7.5 모션 접근성 [P2]

**현재 문제:** `prefers-reduced-motion` 미디어 쿼리 미적용. 모든 사용자에게 동일 애니메이션 강제.

**구현 방안:**

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

```
추가 고려:
- 설정 페이지에 "애니메이션 줄이기" 토글 (OS 설정과 독립적)
- 카운트업 애니메이션 → 즉시 표시
- 페이지 전환 → 즉시 교체
- 스켈레톤 shimmer → 정적 회색 블록
- confetti/파티클 → 표시 안함
```

**우선순위:** P2
**예상 효과:** 전정 장애, 주의력 결핍 등 모션에 민감한 사용자 배려. WCAG 2.3.3 준수.

## 핵심 설계 원칙 (전 팀원 공유)

1. **3초 룰**: 어떤 페이지든 3초 안에 핵심 정보가 보여야 한다.
2. **제로 러닝 커브**: 설명 없이 누르면 작동해야 한다. 텍스트 라벨보다 시각적 단서(아이콘, 색상, 위치)가 먼저.
3. **모바일은 축소가 아니다**: 모바일은 데스크톱의 축소판이 아니라 독립적인 경험. 별도 레이아웃, 별도 플로우.
4. **피드백은 즉각적이어야 한다**: 모든 사용자 액션에 200ms 이내 시각적 반응.
5. **빈 화면은 기회다**: Empty state는 "없음"이 아니라 "다음 행동 안내".
6. **일관성이 세련됨이다**: 같은 패턴은 어디서든 같은 방식으로. 디자인 토큰을 반드시 사용.
7. **성능은 UX다**: 60fps 애니메이션, 100ms 이내 인터랙션 응답, 스켈레톤으로 체감 속도 향상.
